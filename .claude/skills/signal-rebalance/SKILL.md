# Signal Rebalance Pipeline

**已建仓持仓的轻量调整通道**。新建仓走 5 Gate(signal-generation),加减仓只走 3 Gate(本 skill)。Kelly 和瓶颈在开仓时已验证,加减仓时不重做。

## Contract

- **Triggers**: 已有持仓的加仓/减仓/止盈/止损决策 / signal-generation router 路由 / 用户直接说"国茂减仓"
- **Inputs**: 标的代码 + 当前持仓量(从 positions.md) + 当前价 + 调整逻辑(为什么加/减)
- **Outputs**: 完整调整信号行(技术面 + 调整股数 + 净赔率 + 反方 + 信心)+ positions.md 更新建议
- **Calls**: `tool-fallback`(数据失败时)
- **Called by**: `signal-generation` router、`daily-briefing` P3.2(若仅含加减仓)、用户直接

## ⚠️ 强制规则

1. **必先确认是已建仓** — positions.md 里查不到 = 不能用本 skill,必走 signal-generation 完整 5 Gate
2. **3 道闸顺序执行** — Gate 1 / Gate 2.5 / Gate 4,任一失败 = 信号丢弃
3. **加仓 vs 减仓有不同判定逻辑** — 加仓更严(可能追高),减仓更松(锁定盈亏)
4. **止损 / 止盈触发** — 自动减仓,跳过 Gate 1(已是反向信号),仍需 Gate 2.5 校验股数 + Gate 4 反方
5. **🚫 NO GUESSING** — 持仓量/成本价必从 positions.md 读取

## 4 类调整场景

| 场景 | 触发 | Pipeline |
|---|---|---|
| **加仓**(初始仓未满 + 价格更优) | 用户主动 / 跌至更安全位 + 逻辑未变 | Gate 1 + Gate 2.5 + Gate 4 |
| **减仓**(部分止盈 / 风控降仓) | 触止盈位 / 单票 >10% / 主线集中度 >40% | Gate 1 + Gate 2.5 + Gate 4 |
| **止损**(全平/重仓平) | 触发预设止损位 / 主线证伪 | Gate 2.5 + Gate 4(跳 Gate 1)|
| **止盈**(全平/分批平) | 触发预设止盈位 / 估值透支 | Gate 2.5 + Gate 4(跳 Gate 1)|

## 3 道闸(顺序执行)

### Gate 1: 技术面验证(加仓/减仓时必跑;止损止盈跳过)

**数据**:Yahoo `get_historical_stock_prices` period=3mo + `python tools/technical.py`(已建仓不需要 6mo 全量)

**🎭 Stale 守卫**:同 signal-generation Gate 1,Yahoo stale → Playwright 东财。

**必算指标**(精简版,4 项):
- MA20 / MA60 — 趋势 + 加减仓位置参考
- RSI14 — 加仓避超买 / 减仓避超卖
- MACD — 动能方向是否仍支持原逻辑
- 布林 %B — 加仓必 <50,减仓时 >70 加分

**判定**:

| 场景 | 加仓判定 | 减仓判定 |
|---|---|---|
| 🟢 通过 | RSI 30-55 + 价 ≥ MA20 + MACD 未死叉 + %B <50 | RSI >65 / %B >70 / MACD 顶背离 任一 |
| 🟡 警告 | RSI 55-65 + 价远离 MA20 → 减小加仓量 | RSI 30-50 / 趋势仍向上 → 减仓量减半 |
| 🔴 否决 | RSI >70 / MACD 死叉 / %B >80 | 持仓未触位且技术面健康 → 不减 |

**Gate 1 输出**:
```
- 现价 ¥X / MA20 ¥Y / MA60 ¥Z / RSI N / MACD [方向] / %B M%
- vs 原开仓:技术面 [更优 / 一致 / 恶化]
- 加/减仓建议:[绿/黄/红]
```

### Gate 2.5: 可交易性校验(继承 signal-generation Gate 2.5)

**沿用 signal-generation Gate 2.5 的 3 道校验**(详见 [signal-generation SKILL.md](../signal-generation/SKILL.md) Gate 2.5),但参数有调整:

#### 加仓特殊规则
- 加仓后**单票 ≤10%** 硬约束依然生效 — 计算 (现持仓 + 加仓股数) × 现价 / 总资金
- 加仓后**主线集中度 ≤40%** — 必查 positions.md 同主线占比
- 加仓 100 股价值 < ¥1,500 → 不值得(费率 >0.7%)

#### 减仓特殊规则
- 减仓后**剩余持仓 ≥100 股**(否则全平,不留零头)
- 减仓 ≥200 股时,**优先用 100 股分批挂**,避免一次性冲击
- 止盈分档:第一档卖 1/4 - 1/3,留主仓继续涨

**Gate 2.5 输出**:
```
- 调整后总持仓: X 股 / 总价值 ¥A / 单票占比 N% [✅/🔴]
- 主线集中度: M% [✅/🔴]
- 净赔率 b' (加仓时) 或 净止盈 (减仓时): X
- 判定: [PASS / FAIL → 调整股数为 Y / 取消调整]
```

### Gate 4: Devil's Advocate(必写)

继承 signal-generation Gate 4 的 4 类反方,但**调整问题方向**:

| 调整 | 反方问题 |
|---|---|
| **加仓** | 1. 是否在追高?(原成本 vs 现价价差)2. 主线逻辑近期有恶化吗?3. Kelly 还支持加这么多吗(粗算)?|
| **减仓** | 1. 这是真的卖出信号还是恐慌?2. 减少后能否覆盖原赔率?3. 是否被短期波动诱导?|
| **止损** | 1. 止损位是否预设过紧?2. 主线是否真的证伪还是单日错杀?3. 减半 vs 全平哪个对? |
| **止盈** | 1. 还能涨多少 vs 卖出回收的机会成本?2. 第一档卖多少留多少?3. 卖完资金有更好去处吗? |

**判定**:**反方比正方强 → 调整降级或不动**。

## 与 signal-generation 的关系

```
信号判断:这个标的我有持仓吗?(查 positions.md)
    │
    ├─→ 否(新建仓) → signal-generation Gate 1+2+2.5+3+4(完整 5 Gate)
    │
    └─→ 是(已建仓) → signal-rebalance Gate 1+2.5+4(轻量 3 Gate)
                       │
                       ├─→ 加仓 / 减仓 → 全 3 Gate
                       └─→ 止损 / 止盈 → 跳 Gate 1,仅 Gate 2.5+4
```

## Why(为什么需要拆分)

**2026-05-08 至 5/14 教训**:
- 5/12 太极止盈 200 股 @¥11.5 — 用户当时只问"该不该卖",不需要重做 Kelly 和瓶颈分析(已开仓时验证过)。但 signal-generation 强制走 5 Gate,信号生成花了不必要的时间。
- 国茂 5/13 涨停后回调 — 是否减仓只需要"技术面是否反转 + 减仓多少股最合算 + 反方有没有充分理由",**不需要重做"瓶颈定价潜力"评级**。

**轻量 3 Gate 优势**:
- 加减仓决策从 ~30 分钟压到 ~10 分钟
- 跳 Gate 3 瓶颈避免重复劳动(已建仓 = 瓶颈已验证)
- 跳 Gate 2 Kelly 避免误判(Kelly 仅适用于"全新下注",加减仓应基于"已有头寸的边际调整")
- Gate 2.5 仍跑 — 因为持仓变化后 100 股最小单位 / 单票 cap / 集中度都需要重算

## 反模式(严禁)

- ❌ 用 signal-rebalance 做新建仓 — 必须查 positions.md 确认已建仓
- ❌ 加仓时跳 Gate 4 — 加仓最容易追高,反方必须写
- ❌ 止损时跳 Gate 4 — 止损时最容易"卖飞",反方必须问"是真证伪还是短期错杀"
- ❌ 减仓后剩 50 股 — 必须 ≥100 股或全平
- ❌ "之前 Kelly 算过了,现在加仓直接按比例" — 加仓后总仓位变了,Gate 2.5 必重算单票占比
